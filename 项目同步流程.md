# 项目同步流程

本文档用于在拉取最新代码后，快速完成本地环境、依赖、数据库和测试的同步。

## 1. 同步 Python 依赖

本项目推荐使用 `uv`：

```powershell
uv sync
```


## 2. 同步环境变量

确认 `.env` 中关键配置已就绪（至少包含数据库连接）：

- `DATABASE_URL`：必须是 MySQL，格式示例：`mysql+pymysql://user:password@127.0.0.1:3306/app`
- `SECRET_KEY`
- 可选：邮件与会话相关配置（`EMAIL_BACKEND`、`SESSION_*`）

## 3. 同步数据库

应用启动时会根据配置自动初始化库表（`AUTO_CREATE_DB=true` 时）。

如需手动执行 SQL 变更，可按顺序执行 `migrations/` 下脚本：

1. `000_create_databases.sql`
2. `001_create_tables.sql`
3. `002_alter_users_profile_fields.sql`
4. `003_add_preferences_to_users.sql`

## 4. 启动并验证服务

```powershell
uv run main.py
```

默认端口为 `8000`，健康检查接口：`GET /health`。

## 5. 同步后回归测试

先配置测试数据库环境变量：

- `TEST_DATABASE_URL`（必须为 MySQL 连接串）

运行测试：

```powershell
uv run pytest -q
```

## 6. 常见问题排查

1. `DATABASE_URL must use MySQL`：
   检查 `DATABASE_URL` 是否为 `mysql+pymysql://...`。
2. 测试被跳过：
   检查是否设置了 `TEST_DATABASE_URL`。
3. 认证写接口返回 `403 csrf token missing or invalid`：
   登录后在请求头中附带 `X-CSRF-Token`。

## 7. 推荐同步顺序

```powershell
uv sync
uv run main.py
uv run pytest -q
```

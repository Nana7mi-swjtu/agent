# 项目同步流程

本文档用于拉取最新代码后，快速完成本地环境、依赖、数据库、前后端联调与 RAG/Agent 能力同步。

## 1. 同步依赖

后端依赖（推荐 `uv`）：

```powershell
uv sync
```

前端依赖：

```powershell
npm --prefix frontUI install
```

## 2. 同步环境变量

以 `.env.example` 和 `frontUI/.env.example` 为基线补全本地配置。

### 2.1 后端 `.env`

基础项：

- `DATABASE_URL`（必须 MySQL，示例：`mysql+pymysql://user:password@127.0.0.1:3306/app`）
- `TEST_DATABASE_URL`（测试库，MySQL）
- `SECRET_KEY`
- `AUTO_CREATE_DB=true`（开发环境建议开启）

CORS：

- `CORS_ENABLED=true`
- `CORS_ALLOW_CREDENTIALS=true`
- `CORS_ALLOWED_ORIGINS=http://127.0.0.1:5173,http://localhost:5173`
- `CORS_ALLOWED_HEADERS` 需包含 `X-CSRF-Token`

Agent：

- `AI_PROVIDER`
- `AI_MODEL`
- `AI_API_KEY`
- `AI_BASE_URL`
- `AI_TIMEOUT_SECONDS=30`

RAG / Chroma：

- `RAG_ENABLED=true|false`
- `RAG_DEBUG_VISUALIZATION_ENABLED=true|false`（控制调试接口与调试数据透传）
- `RAG_VECTOR_PROVIDER=chromadb`
- `RAG_EMBEDDER_PROVIDER`
- `RAG_EMBEDDING_MODEL`
- `RAG_EMBEDDING_VERSION`
- `RAG_EMBEDDING_DIMENSION`
- `RAG_EMBEDDING_API_KEY`
- `RAG_EMBEDDING_BASE_URL`
- `RAG_RERANKER_PROVIDER`
- `RAG_RERANKER_MODEL`
- `RAG_RERANKER_API_KEY`
- `RAG_RERANKER_BASE_URL`
- `RAG_RETRIEVAL_TOP_K=5`
- `RAG_RETRIEVAL_SCORE_THRESHOLD=0.0`（过滤阈值，越高越严格）
- `RAG_ALLOWED_FILE_TYPES=pdf,docx,md,txt,html,csv`
- `RAG_UPLOAD_DIR`
- `RAG_AUTO_INDEX_ON_UPLOAD=true`
- `RAG_CHUNK_SIZE=1200`
- `RAG_CHUNK_OVERLAP=150`
- `RAG_CHUNK_STRATEGY_DEFAULT=paragraph`
- `RAG_CHUNK_STRATEGY_ALLOWED=paragraph,semantic_llm`
- `RAG_CHUNK_FALLBACK_STRATEGY=paragraph`
- `RAG_CHROMADB_PERSIST_DIR`
- `RAG_CHROMADB_COLLECTION_PREFIX=rag`

### 2.2 前端 `frontUI/.env.development`

- `VITE_API_BASE_URL=http://127.0.0.1:8000`
- `VITE_RAG_DEBUG_VISUALIZATION_ENABLED=true|false`（前端是否展示调试面板）

## 3. 同步数据库

应用启动时会自动执行：

1. 数据库存在性检查/创建（MySQL）
2. ORM `create_all`
3. 关键表字段补齐（含 RAG 文档/分块/任务相关扩展字段）

如需手动初始化，按 `migrations/` 顺序执行到最新版本，当前包括：

1. `000_create_databases.sql`
2. `001_create_tables.sql`
3. `002_alter_users_profile_fields.sql`
4. `003_add_preferences_to_users.sql`
5. `004_add_rag_tables.sql`
6. `005_add_rag_chunking_strategy_fields.sql`

## 4. 启动服务

启动后端：

```powershell
uv run main.py
```

健康检查：

- `GET /health` 返回 `{"ok": true}`

启动前端：

```powershell
npm --prefix frontUI run dev
```

默认访问：`http://127.0.0.1:5173/#/login`（端口被占用时 Vite 会自动顺延）。

## 5. 联调关键链路

1. 登录成功，浏览器携带会话 Cookie
2. 进入工作台并选择角色（写入 `users.preferences.workspace.role`）
3. 聊天接口返回：
   - 正常：`200`，包含 `data.role / data.systemPrompt / data.reply / data.citations / data.noEvidence`
   - 当 `RAG_DEBUG_VISUALIZATION_ENABLED=true`：额外包含 `data.debug`
4. Agent 配置异常时：`502` + `{"ok": false, "error": "agent service unavailable"}`

## 6. RAG 接口联调（`RAG_ENABLED=true`）

核心链路：

1. `POST /api/rag/upload` 上传文档（`multipart/form-data`，字段 `file`）
2. `POST /api/rag/index` 触发索引
3. `GET /api/rag/jobs/<job_id>?workspaceId=...` 查询任务状态
4. `POST /api/rag/search` 检索验证（`query/topK/filters`）
5. `GET /api/rag/documents?workspaceId=...` 查看文档状态

调试接口（需 `RAG_DEBUG_VISUALIZATION_ENABLED=true`）：

- `GET /api/rag/debug?workspaceId=...`
- `GET /api/rag/embedding?workspaceId=...&chunkId=...&sampleSize=16&full=false`

说明：

- 所有写接口需要登录态 + `X-CSRF-Token`
- 当 `RAG_ENABLED=false` 时，`/api/rag/*` 返回 `404`（`rag feature is disabled`）

## 7. 命令行查看 Chroma 向量

可直接运行脚本 `view_chroma_vectors.py`：

```powershell
uv run python view_chroma_vectors.py --list-only
uv run python view_chroma_vectors.py --sample-size 16
uv run python view_chroma_vectors.py --collection <collection_name> --chunk-id <chunk_id> --sample-size 16
uv run python view_chroma_vectors.py --collection <collection_name> --chunk-id <chunk_id> --full
```

更多参数见：`docs/chroma_vector_commands.md`。

## 8. 回归测试

```powershell
uv run pytest -q
npm --prefix frontUI run build
```

## 9. 常见问题排查

1. `DATABASE_URL must use MySQL`  
   检查 `DATABASE_URL` 是否是 `mysql+pymysql://...`。

2. 测试被跳过  
   检查 `TEST_DATABASE_URL` 是否设置且为 MySQL。

3. `403 csrf token missing or invalid`  
   检查登录态、`X-CSRF-Token`、前端 `credentials: include`。

4. 聊天返回 `agent service unavailable`  
   检查 `AI_MODEL / AI_API_KEY / AI_BASE_URL`。

5. 检索命中少或无命中  
   先确认索引任务 `done`，再调低 `RAG_RETRIEVAL_SCORE_THRESHOLD`。

6. 调试接口返回 `rag debug visualization is disabled`  
   检查 `RAG_DEBUG_VISUALIZATION_ENABLED=true` 并重启后端。

7. 向量脚本提示找不到目录  
   检查 `RAG_CHROMADB_PERSIST_DIR` 是否正确，或使用 `--persist-dir` 手动指定。

## 10. 推荐同步顺序（标准）

```powershell
uv sync
npm --prefix frontUI install
uv run main.py
npm --prefix frontUI run dev
uv run pytest -q
npm --prefix frontUI run build
```

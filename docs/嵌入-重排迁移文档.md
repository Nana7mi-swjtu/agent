# RAG 嵌入与重排 Provider 迁移记录（2026-03-26）

## 1. 目标

本次改动的目标是：

- 移除 RAG 中 embedder / reranker 的默认回退路径（不再允许“未配置也能跑”）。
- 将嵌入能力切换为 `langchain_community` 的 `DashScopeEmbeddings`。
- 将重排能力改为显式 provider 模式（生产可用 `openai-compatible`，测试使用 `fake`）。

---

## 2. 行为变化（重要）

### 2.1 配置从“可缺省”变为“必须显式配置”

现在如果未配置以下字段，会直接抛 `RAGConfigurationError`：

- `RAG_EMBEDDER_PROVIDER`
- `RAG_RERANKER_PROVIDER`

也就是说，系统不再默认使用 deterministic/noop 作为 embedder/reranker。

### 2.2 嵌入模型可通过 provider 显式切换

生产推荐配置（Qwen + openai-compatible）：

- `RAG_EMBEDDER_PROVIDER=openai-compatible`
- `RAG_EMBEDDING_MODEL=Qwen-Embedding`（或你实际使用的 Qwen embedding 模型）
- `RAG_EMBEDDING_API_KEY=<your-key>`
- `RAG_EMBEDDING_BASE_URL=<your-openai-compatible-base-url>`

### 2.3 重排模型显式配置

生产推荐配置（Qwen + openai-compatible）：

- `RAG_RERANKER_PROVIDER=openai-compatible`
- `RAG_RERANKER_MODEL=qwen-reranker-v1`
- `RAG_RERANKER_API_KEY=<your-key>`
- `RAG_RERANKER_BASE_URL=<your-openai-compatible-base-url>`

## 5. 使用说明（你当前场景）

你希望“只使用提供的模型”，请在 `.env` 至少配置：

```env
RAG_EMBEDDER_PROVIDER=openai-compatible
RAG_EMBEDDING_MODEL=Qwen-Embedding
RAG_EMBEDDING_API_KEY=你的密钥
RAG_EMBEDDING_BASE_URL=你的兼容OpenAI地址

RAG_RERANKER_PROVIDER=openai-compatible
RAG_RERANKER_MODEL=qwen-reranker-v1
RAG_RERANKER_API_KEY=你的密钥
RAG_RERANKER_BASE_URL=你的兼容OpenAI地址
```

若这几项缺失，RAG 在检索/索引阶段会按预期报配置错误，而不是静默回退。
